# telos

> ビジネスパーソンの「散らかった仕事の情報」を、判断と行動につながる形に整理する web アプリ。
> τέλος（テロス）= ギリシャ語で「目的・最終目標」。すべての整理は目的のためにある。

## なぜ作るのか

仕事の情報は放っておくと3つの形で散らかる。

1. **やるべきことが頭の中にだけある** — リリース前の確認事項を記憶と経験に頼ると、必ず抜ける。外科手術ですらチェックリストの導入で死亡率が半減した（Haynes et al., 2009）。ソフトウェアリリースも同じ構造の問題を抱えている。
2. **報告が「起きたことの羅列」になる** — 進捗報告は、聞き手（上司・経営層）が知りたい「結論」から始まらないと伝わらない。結論を先に置く構造（ピラミッド原則）は、意思決定者の認知の流れに合っている。
3. **会話が流れて、本当の目的が埋もれる** — 打ち合わせやチャットでは要望・不満が大量に出るが、その奥にある「本当に叶えたいこと」は明示されない。表層の要望のまま作ると、誰も使わないものができる。

詳細なエビデンスは [docs/concept.md](docs/concept.md)。

## コアコンセプト

> **情報を「記録」で終わらせず、目的（telos）に向かう道具に変える。**

3つのツールは全部この軸から導かれる：

- **Checklist** — 「やるべきこと」を記憶から外に出し、リリース判断の道具にする
- **Slides** — 「起きたこと」を結論ファーストに組み替え、報告の道具にする
- **Goals** — 「言われたこと」を掘り下げ、本当の目的を見つける道具にする

## 構成

- `app/` — Vite + TypeScript（フレームワークなし、クライアントサイドのみ、localStorage 永続化）

```bash
cd app
npm install
npm run dev
```
