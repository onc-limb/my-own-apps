# telos — ロードマップ

## 設計哲学

- **クライアントサイド完結**: サーバーなし。データは localStorage。個人の仕事情報を外部に送らない。
- **フレームワークなし**: Vite + vanilla TypeScript。依存を最小にし、長期間メンテなしでも動く状態を保つ（image-comp と同方針）。
- **構造をテンプレートで強制する**: 自由入力より「正しい構造に誘導するフォーム」。エビデンスのある型（チェックリスト / ピラミッド原則 / 5 Whys + JTBD）をそのまま UI にする。

## 技術スタック

- pnpm ワークスペース（`app` / `server`）
- TypeScript 5.x + Vite 6.x（フロント）
- 永続化: localStorage（キー: `telos:*`）。JSON でインポート/エクスポート可能
- ルーティング: ハッシュベースの自前ルーター
- PDF 出力: ブラウザ印刷 + print CSS
- LLM 連携（任意）: ローカルブリッジ（Node 標準 http）経由。`claude -p`（サブスク）/ LiteLLM 互換 API の2形態。API キーはサーバ側に隔離

## デザイン方針

- **アプリ本体はダークなエンジニア系**（mono フォントのアクセント、グラデーション、グリッド感）
- **成果物（Slides / Brief）は白い「紙」**。届ける相手が読むものはごちゃごちゃさせず、余白とコントラストで見やすく

## フェーズ

### Phase 1 — MVP（完了）

3ツール（Checklist / Slides / Goals）の基本動作。詳細は [docs/features.md](../docs/features.md)。

**完成基準:**
- [x] リリースチェックリストをテンプレートから作成し、チェック・追加・削除・進捗表示ができる
- [x] フォーム入力から進捗報告スライドを生成し、プレゼン表示と印刷（PDF）ができる
- [x] 会話ログから要望発言を抽出し、5 Whys で掘り下げて JTBD 形式の目的文を保存できる
- [x] ビジネス説明用の一枚資料（Brief）を作成し、PDF 出力できる
- [x] 自分の価値を言語化したバリューステートメント（Positioning）を作成・PDF 出力できる
- [x] 振り返りから継続提案（Retrospective）を生成し、クライアント向け一枚を PDF 出力できる
- [x] 見積（Estimate・工数/金額レンジ）、提案書（Proposal）、議事録（Decisions）、変更管理（Changes）を作成・PDF 出力できる
- [x] 全10ツールが案件ライフサイクル（入口→進行→報告→継続）に沿って並ぶ
- [x] 全データを JSON でインポート / エクスポートできる
- [x] LLM 連携（任意）を `claude -p` / API の両形態で有効化でき、Goals・Brief の AI 支援が動く
- [x] すべてのデータがリロード後も保持される（localStorage）
- [x] `pnpm build` が通り、ブリッジサーバが型チェックを通る
