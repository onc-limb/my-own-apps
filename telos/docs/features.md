# telos — 機能一覧

すべての機能は「情報を記録で終わらせず、目的に向かう道具に変える」というコンセプトから導かれる。

## Phase 1 — MVP（実装済み）

### Checklist — リリースチェックリスト

| 機能 | 説明 |
|------|------|
| テンプレートから作成 | 7カテゴリ（企画・要件 / 開発・品質 / セキュリティ / 運用準備 / ドキュメント / 告知・サポート / リリース後）約25項目の標準リリースチェックリストから新規作成 |
| 項目のチェック / 進捗表示 | カテゴリ別 + 全体の進捗バー。リリース判定の状態が一目で分かる |
| 項目の追加・削除 | プロジェクト固有の確認事項を追加できる |
| 複数チェックリスト管理 | リリースごとに独立したインスタンスを保持 |

### Slides — 進捗報告スライド

| 機能 | 説明 |
|------|------|
| フォーム入力 → スライド生成 | プロジェクト名・期間・状況（順調/注意/危険）・サマリー・成果・課題・次アクションを入力すると、結論ファースト構造のスライドが生成される |
| プレゼンモード | スライドを1枚ずつ表示。キーボード（←/→）で操作 |
| PDF 出力 | ブラウザ印刷で1スライド=1ページの PDF に出力（印刷 CSS） |
| 報告書の保存・複製 | 過去の報告を保存し、次回の報告のベースにできる |

### Goals — 目的の発見

| 機能 | 説明 |
|------|------|
| 会話ログの貼り付け・分析 | 議事録・チャットログを貼り付けると、要望・不満・目的を示す発言を自動抽出（「〜したい」「〜が課題」等のマーカー検出）し、頻出キーワードを表示 |
| 5 Whys ガイド | 抽出された発言から1つ選び、「なぜそれが重要か」を最大5段階掘り下げるガイド付きフォーム |
| JTBD 形式の目的文 | 掘り下げの結果を「（状況）のとき、（動機）したい。そうすれば（期待する進歩）」の形式に着地させる |
| セッション保存 | 分析と掘り下げの過程を保存し、後から見返せる |

## Phase 2 — 候補（未実装）
