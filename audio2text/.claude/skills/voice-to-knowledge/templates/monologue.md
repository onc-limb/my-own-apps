# Source Note Template: Monologue / Thought Memo

自分の独り言・思考を**思考の流れを残しつつ1ファイルで構造化する** ためのテンプレート。

## 抽出時の基本方針

- 結論が出ていない / 矛盾している思考もそのまま残すことに価値がある
- 「考えていたテーマ」を `## 話題の流れ` でテーマ単位に整理
- 「次に動くべきこと」を `## タスク` に明示。独白は行動につながらないと無意味になりがち
- 感情の動き（イライラ、納得、興奮）も `## 学び・気付き` に
- 文意が不明な部分は無理に整形せず、`## 引用` で残す
- アイデア・問い・気付きが多くなる傾向。決定・事実は少なめでも良い

## 出力フォーマット

```markdown
---
title: <思考のテーマを表す短いタイトル>
source_audio: <元音声ファイル名>
source_transcript: <transcript .md ファイル名>
recorded_at: <YYYY-MM-DD>
processed_at: <ISO8601>
category: monologue
preset: <使用したプリセット名>
language: <ja|en|...>
duration: <HH:MM:SS>
mood: <例: focused / restless / excited / frustrated>
project: <プロジェクト名>
tags:
  - <キーワード>
---

# <Title>

## 要約 (TL;DR)

<必須。2〜3文で「何について考えて、どこに着地したか / 着地しなかったか」>

## 話題の流れ

<論理の流れ。途中で方向が変わったらその転換点を明示。>

1. <最初の問題意識>
2. <仮説>
3. <反証や疑問>
4. <暫定的な着地点>

## 決定事項

<暫定的に決めたこと。独白では稀。>

- **<決定>**: <内容>

## タスク

<自分への TODO。行動につなげる。>

- [ ] <内容>

## アイデア

<採否未定の発想・仮説。>

- **<アイデア>**: <内容>

## 学び・気付き

<自分の認識が変わった瞬間、感情の動きも含む。>

- **<気付き>**: <内容>

## 知識・事実

<考察中に確認した事実。>

- ...

## 未解決の問い

<考え終わらなかったこと。次の思考のフック。>

- **<問い>**: <背景>

## 用語・概念

<自分なりに定義した用語。>

- **<用語>**: <定義>

## キーワード

`<keyword1>` `<keyword2>` ...

## 引用 / 印象的な独白

<3〜5個。考えのクリスタルになっている発言。>

- `[HH:MM:SS]` <発言>

## 関連

- [[sources/<YYYY-MM>/<related-file>]]
```

## 注意

- 該当する内容がないセクションは見出しごと省略してよい。`## 要約 (TL;DR)` だけは必須
- 意味が通らない部分は「一貫性のために改変する」のではなく `[判読不能]` 等で残す
- 推測が混じる場合は `<!-- 推測: ... -->` で明記
