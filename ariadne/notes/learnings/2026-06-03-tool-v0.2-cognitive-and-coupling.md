# 完成ツール v0.2: 三項対応・認知的複雑度・結合度メトリクス（ABC）

## コンテキスト
完成ツール (`tool/`) に、ロードマップ②③にあたる機能をまとめて追加した
（試すプロジェクトが手元に無いとのことで、A/B/C を一気に実装）。

## A: 三項演算子の算入 + 共通 Lexer
- 字句レベルで `?` を判別: `?.`（オプショナルチェーン）/`??`（nullish）/
  `x?:`（省略可能マーカー、空白を飛ばして次が `:` なら）を除外し、残りを三項として算入。
- 複雑度計測を正規表現ベースから **トークン列ベース (`lib/lexer.ml`)** に統一。
  `re` の部分文字列カウントより誤検出が減り、B/C と基盤を共有できる。

## B: 認知的複雑度（SonarSource）
近似実装。原則は B1 基本加点 / B2 ネスト加重 / B3 読みやすい要素は加点しない。
- ネスト単位は波括弧で近似。`{` に来たとき直前の「控え」が制御構造か関数かで
  ネストスコープか判定。
- **トップレベル関数本体はネスト 0**、ネストした関数のみネスト +1。
- `else if` は else 側で +1 のみ（if 側は再加点・ネスト加点しない）。
- `switch` は 1 回、`case` は数えない。`do`-`while` の末尾 `while` はスコープ
  opener を覚えてスキップ。

### つまずき: 論理演算子の連続が途切れた
`a && b && c` が 1 になるべきところ 2 になった。原因は **オペランドの識別子 Word でも
`prev_logical` をリセットしていた** こと。`a` `b` `c` で毎回リセットされ、`&&` の連続が
分断されていた。修正: prev_logical のリセットは **制御キーワード等の式境界のみ** に限定し、
オペランドではリセットしない。
→ 学び: 「列（sequence）」を判定するロジックでは、列の構成要素そのものでリセットしてはいけない。

## C: アーキテクチャ指標（結合度）
- `lib/imports.ml`: コメントだけ除去（文字列は残す）→ 正規表現で import 指定子と
  export の抽象/具体を抽出。Tokenizer.scan は文字列も潰すので import 用には使えない点に注意。
- `lib/graph.ml`: 相対指定子を `Fpath.normalize` で正規化したパスに解決し、
  `index.*` と拡張子補完を試す。バレ指定子（`react` 等）と解決不能は外部依存。
  - Ca/Ce → 不安定度 I=Ce/(Ca+Ce)、抽象度 A、主系列距離 D=|A+I-1|。
  - I の Ce は内部依存のみ（外部は別カラム EXT）。Ca は内部からの被依存のみ
    （プロジェクト外からの被依存は観測不能なため）。

## 動作確認（ドッグフーディング）
小さな3モジュール構成で検証し、手計算と一致:
- `types.ts`: A=1.0, Ca=2, I=0, D=0（安定した抽象 = 理想）
- `repo.ts`: I=0.5, A=0, D=0.5（やや主系列から外れ＝改善余地のシグナル）
- `service.ts`: I=1.0, A=0, D=0, EXT=1（不安定なトップ層 = 妥当）
