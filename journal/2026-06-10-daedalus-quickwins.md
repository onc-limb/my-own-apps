# 2026-06-10 daedalus クイックウィン実装 — plan サマリ & tfsec 必須化

## 指示（R-001）

機能アイデア一覧（`daedalus/docs/feature-ideas.md`）の提示に対し、ユーザーから
「おすすめから着手して」。→ 推奨していたクイックウィン2つを実装した:

1. **plan の人間向けサマリ**（ユーザー目線）
2. **tfsec セキュリティスキャン必須化**（エンジニア/管理者目線）

## 設計判断と理由

- **plan サマリは LLM に要約させず、正規表現の決定的パースで作る**
  （新モジュール `terraform_output.py`）。
  理由: 追加コスト・遅延ゼロ、再現性があり、誤要約のリスクがない。terraform の
  `will be created/updated in-place/destroyed/must be replaced` と `Plan: X to add...` 行は
  安定したマーカーで、ANSI を剥がせば確実に拾える。LLM 要約は「構造化 plan(-json)」導入後に
  必要なら検討。
- **tfsec ゲートはステートマシン方式**: クリーンなスキャン（CRITICAL/HIGH ゼロ）で解錠、
  `.tf` の編集（Write/Edit フック、Bash の `>`/`sed -i`/`tee` 書き込み含む）で再施錠。
  施錠中の `terraform apply` は deny し、理由文で Claude を tfsec 実行へ誘導する。
  理由: 「プロンプトでお願いする」だけでは強制にならない。ガードレールと同じ deny 機構に
  乗せることで機械的に担保される。
- **人間承認より先にスキャンゲートを評価**: レビュアーには「スキャン済みの plan」だけが届く。
- **tfsec 未導入なら警告して自動無効化**（`--no-security-scan` で明示OFFも可）。
  理由: 必須化のデフォルトONを保ちつつ、未導入環境で run が永久に詰まる事態を避ける。
  ブロックでなく警告＋導入案内（setup-guide に追記）で導入へ誘導する。
- **遮断はCRITICAL/HIGHのみ**。MEDIUM/LOW は「直せるなら直す、無理なら最終サマリで報告」を
  プロンプトで指示（一般的なポリシー運用に合わせた）。

## どこで効くか

- 「エージェントに必須手順を守らせたい」場面の型: プロンプト指示＋**フックの deny ゲート＋
  状態無効化**の3点セット。今後のポリシーガードレール（OPA）や予算ガードも同じ型で実装できる。

## 宿題

- 構造化 plan（`terraform show -json`）への移行で正規表現パースを置き換える
- 残りのクイックウィン: 通知（Slack）/ spec テンプレギャラリー
- tflint / checkov の追加（tfsec と同じゲート機構に乗せる）
