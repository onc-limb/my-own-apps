# marginalia ロードマップ

> **コンセプト:** 本を「厚みと位置を持つ物体」として扱い、紙の身体感覚（空間記憶）を取り戻す iPhone リーダー。
> 詳細は [../docs/concept.md](../docs/concept.md)、機能の全体像は [../docs/features.md](../docs/features.md) を参照。

## 設計哲学

- 機能を盛るのではなく、**「厚み・小口・付箋」という1本の軸**から全機能を導く。
- **位置の安定を最優先**（空間記憶の前提）。取り込み時にページを確定し、以後不変にする。
- PDF と EPUB を「動かないページの集合」という**1つの内部表現に正規化**する。

## 技術スタック（決定）

| レイヤー | 採用 | 補足 |
|----------|------|------|
| 言語・UI | Swift + SwiftUI（要所で UIKit ブリッジ） | iPhone 専用。触覚・PDFKit をネイティブでフル活用 |
| PDF | PDFKit（純正） | 表示・テキスト抽出・注釈・cropBox によるトリミング |
| EPUB | フェーズ後半で Readium 等を評価 | MVP は PDF のみ |
| 触覚 | Core Haptics | 付箋の種類ごとに触覚パターンを変える |
| 永続化 | SwiftData | 付箋・進捗・本棚メタ。本体ファイルはファイルシステム |
| 取り込み | UIDocumentPicker / 共有シート | ファイル App 連携 |

- 対応 OS: **iOS 17 以上**（SwiftData 採用のため）。開発: Xcode（macOS）。
- 開発者は iOS/Swift がほぼ初めて。セットアップ・基礎から段階的に進める。

## フェーズ概要

| Phase | テーマ | ゴール |
|-------|--------|--------|
| 1 (MVP) | 1冊の読書体験 | 取り込み → 表示 → 小口 → 色分け付箋 → 掴んでジャンプ → 進捗記録。パラパラ + 触覚まで。 |
| 2 | 横断と振り返り | 本棚（背表紙の進捗一覧）・学びノート（付箋の一覧ビュー）。 |
| 3 | 発展 | 俯瞰マップ・全書籍横断検索・エクスポート/復習・読みやすさモード。 |

## Phase 1 (MVP)

1冊の PDF を取り込んで「紙のように」読める状態をゴールとする。実装ステップ・データモデル・完成基準の詳細は [phase1-mvp.md](phase1-mvp.md) を参照。

スコープ判断: MVP は **PDF のみ**。PDF は固定ページなので位置は安定（空間記憶の前提を満たす）。文字サイズは余白トリミング + ズームで対応し、**テキスト再レイアウトはフェーズ後半**に回す。
