# Phase 1 (MVP) — 1冊の読書体験

> **ゴール:** PDF を1冊取り込んで「紙のように」読める。小口で進捗が見え、掴んでジャンプでき、種類別の付箋を刺せ、パラパラめくりに触覚が返り、続きから読める。

## スコープ

- 対象は **PDF のみ**（EPUB はフェーズ後半）。
- PDF は**固定ページのまま**扱い、位置を安定させる（空間記憶の前提を満たす）。
- 文字サイズは**余白トリミング + ズーム**で「見かけを大きく」して対応。**テキスト再レイアウトはやらない**（実装難度が高く、EPUB 対応と合わせてフェーズ後半へ）。

## 技術

- Swift + SwiftUI（PDF 表示・小口描画・ジェスチャーは `UIViewRepresentable` で UIKit / PDFKit を埋める）
- PDFKit / Core Haptics / SwiftData / UIDocumentPicker
- iOS 17+ / Xcode

## 実装ステップ

各ステップで動作確認できる垂直スライス。上から順に積み上げる。

### Step 1: プロジェクト土台 + PDF 表示
- Xcode で新規 SwiftUI プロジェクト（iOS 17+）。
- PDFKit の `PDFView` を `UIViewRepresentable` でラップ。
- バンドルに入れたサンプル PDF を1つ表示。
- **確認:** アプリ起動 → PDF が表示されページ送りできる。

### Step 2: 取り込み + ライブラリ
- `UIDocumentPicker` で PDF を選んでアプリ内に取り込む。
- SwiftData の `Book` モデルを定義して保存。
- ライブラリ画面で一覧表示 → タップで開く。
- **確認:** 端末内の PDF を取り込み、一覧から開ける。

### Step 3: 進捗記録 + 小口（厚みゲージ）
- 現在ページ / 総ページから進捗を計算し `Book` に保存（最後に読んだページ）。
- 画面右端に小口（厚みゲージ）を SwiftUI で描画。既読/未読を厚みで表現。
- **確認:** 読み進めると小口が変化し、再オープン時に続きから開く。

### Step 4: 小口を掴んでジャンプ
- 小口へのドラッグジェスチャーで `PDFView` のページを移動。
- 位置に対応してジャンプ（真ん中を掴めば真ん中へ）。
- **確認:** 小口を掴んで任意の位置へ飛べる。

### Step 5: 付箋（ハイライト / メモ）
- PDF のテキスト選択 → ハイライト作成、種類（色）を選ぶ。
- `Annotation` モデルを定義して保存（ページ + 矩形 + 種類 + メモ）。
- 小口上に付箋ピンを位置対応で描画。タップでその位置へ飛ぶ。
- **確認:** 付箋を作ると小口に表示され、タップで飛べる。

### Step 6: パラパラ + 触覚
- 小口の高速ドラッグでサムネのスキミング表示。
- Core Haptics: ページ送りの tick（速度連動）、付箋通過で種類別の触覚。
- **確認:** パラパラめくると触覚が返り、付箋通過で種類ごとに違う感触になる。

### Step 7: 仕上げ
- 余白トリミング + ズームでの文字サイズ調整。
- 設定: 触覚オン/オフ、効果音（紙めくり）オン/オフ。
- **確認:** 設定が反映され、トリミングで読みやすくなる。

## データモデル（SwiftData）

### Book
| フィールド | 型 | 意味 |
|-----------|-----|------|
| title | String | 表示名 |
| fileRef | — | ファイル参照（コピー先パス or セキュリティスコープブックマーク） |
| addedAt | Date | 追加日 |
| totalPages | Int | 総ページ数 |
| lastReadPage | Int | 最後に読んだページ |
| progress | Double | 進捗（0–1） |

### Annotation（付箋）
| フィールド | 型 | 意味 |
|-----------|-----|------|
| book | Book | 所属（関連） |
| category | enum | 種類（重要 / 疑問 / 引用 / あとで読む） |
| kind | enum | ハイライト / メモ |
| pageIndex | Int | ページ番号 |
| rect | CGRect 相当 | ページ内の位置 |
| note | String? | メモ本文 |
| createdAt | Date | 作成日 |

> MVP では付箋の位置を「ページ番号 + ページ内の矩形」で持つ。PDF は固定ページなのでこれでズレない。本格的なテキストアンカーは再レイアウト導入時（フェーズ後半）に対応。

## 完成基準

- [ ] PDF を取り込んでライブラリで管理できる
- [ ] PDF を表示しページ送りできる
- [ ] 右端の小口で進捗（厚み）が常に分かる
- [ ] 小口を掴んで任意の位置へジャンプできる（真ん中を開ける）
- [ ] 種類別の色付き付箋を作成・保存・表示でき、タップで飛べる
- [ ] パラパラ時に触覚が返る（付箋通過で種類別）
- [ ] 続きから読める（進捗の永続化）
- [ ] 触覚 / 効果音のオン・オフ、余白トリミングが効く

## 未決・調査事項

- `PDFView` のページ移動 API とカスタムジェスチャーの結合方法。
- Core Haptics の連続パターン（高速パラパラ時の間引き）。
- ファイル取り込みは「コピー」か「ブックマーク参照」か（容量・権限のトレードオフ）。
